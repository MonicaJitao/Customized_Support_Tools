# Customized Support Tools 文件说明

## 概述

本工具是面向承兑业务团队的浏览器端计算支持平台，供客户经理在日常业务中快速完成保证金比例测算、调价方案制定及收益分析。工具以单一 HTML 文件形式交付，无需安装，无需后端服务器，打开即用。

---

## 功能模块

### 承兑保证金比例计算器
支持价格与保证金比例的双向换算。客户经理可根据已知条件灵活选择计算方向，结果可导出为 PNG 图片，便于存档或向客户展示。

### 一户一策调价计算器
针对单一客户的调价决策场景。区分 520 客户与普通客户，输入相关参数后自动生成价格-余额曲线，辅助判断合理调价区间。当报价低于风险阈值时，页面会显示预警提示。

### 承兑-定期方案测算
用于向客户提供双方案对比。方案 A 与方案 B 并列展示，收益数据一览无余，并根据测算结果自动标注所需审批层级（团队长或部门长），确保流程合规。

### 承兑-收益打平测算
在保证金比例调整场景下，计算达到收益打平所需的目标比例。支持活期、3 个月、6 个月三种存款类型，结合历史敞口与保证金数据，以曲线形式呈现打平点位置。

### 参数配置中心（管理员专用）
集中管理所有计算器的底层参数，包括业务成本、风险成本、信用 FTP 等。每次参数变更均保留历史版本，支持随时回溯应用。内置自定义公式构建器，采用按钮式 Token 拼接方式，无需编写代码，亦不存在脚本注入风险。

### 使用统计（管理员专用）
记录各工具的使用频次及操作明细，保留最近 200 条日志，支持导出与清空。

| 角色 | 可访问模块 |
|------|-----------|
| 管理员 | 全部功能，含参数配置与使用统计 |

## 数据存储

### 本地存储（localStorage）

业务参数、历史配置、登录状态等数据均存储于浏览器本地，断网环境下可正常使用。数据与浏览器绑定，更换设备后需重新配置。

| 存储键 | 内容 |
|--------|------|
| `appSession` | 当前登录信息（角色、操作员、团队、区域） |
| `currentFTP` | 当前 FTP 配置 |
| `ftpHistory` | FTP 变更历史 |
| `priceAdjustConfig` | 调价计算器参数 |
| `commonConfig` | 公共参数（业务成本、风险成本、信用 FTP） |
| `chengduiConfig` | 承兑工具参数 |
| `usageStats` | 本地使用统计缓存 |
| `customFormulas` | 自定义公式（Token 序列） |

### 云端日志（Firebase Firestore）

每次工具调用时，系统会向 Google Firebase 的 Firestore 数据库异步写入一条操作日志，用于管理员统计分析。若网络不可用，日志自动降级至本地存储，不影响计算功能的正常使用。

**日志字段：**

| 字段 | 说明 |
|------|------|
| tool | 工具名称 |
| operator | 操作员姓名 |
| team | 所属团队 |
| region | 所属区域 |
| time | 操作时间戳 |

## 公网访问架构

本工具通过 **DigitalPlat 免费子域名 + Cloudflare** 的组合实现公网可访问，整体为纯静态托管方案，无需自建服务器。

### 部署链路

```
用户浏览器
    │
    ▼
Cloudflare（DNS 解析 + CDN + HTTPS）
    │
    ▼
GitHub Pages（静态文件托管，index.html）
    │
    ▼
DigitalPlat 子域名（*.digitalplat.org，免费注册）
```

### 各层职责

**DigitalPlat 子域名**
DigitalPlat 提供免费的二级域名注册服务（如 `xxx.digitalplat.org`），用于替代 GitHub Pages 默认的 `*.github.io` 域名，使访问地址更简洁、易记。域名的 DNS 解析权交由 Cloudflare 托管。

**Cloudflare**
承担以下核心职责：
- **DNS 解析**：将子域名指向 GitHub Pages 的服务器 IP
- **HTTPS 证书**：自动签发并续期 TLS 证书，确保访问全程加密
- **CDN 加速**：在全球边缘节点缓存静态资源，降低访问延迟
- **安全防护**：提供基础的 DDoS 防护与恶意请求过滤

**GitHub Pages**
作为静态文件的实际托管平台，将仓库中的 `index.html` 直接对外提供服务，无需额外的 Web 服务器配置。

### 访问流程

1. 用户在浏览器输入子域名
2. Cloudflare DNS 将请求解析至 GitHub Pages 服务器
3. GitHub Pages 返回 `index.html` 文件
4. 浏览器在本地执行全部计算逻辑，按需向 Firebase Firestore 写入日志

整个链路均为免费服务，运维成本为零，适合内部工具的轻量化部署场景。

## 技术实现

- **交付形式**：单一 HTML 文件，包含全部样式与逻辑，无构建依赖
- **图表渲染**：基于浏览器原生 Canvas API 实现，支持高分辨率屏幕适配
- **公式引擎**：自研递归下降解析器，支持四则运算、开方、取整等运算，不使用 `eval`
- **响应式布局**：CSS Grid + Flexbox，兼容桌面与移动端，移动端采用抽屉式侧边栏
- **打印支持**：内置 A4 横向打印样式，可直接通过浏览器打印功能输出报告
