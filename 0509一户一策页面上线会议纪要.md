## 一、会议目标

本次会议聚焦 **“一户一策页面如何上线”**，核心要解决三个问题：

1. 采用哪种上线方式最合适
2. 是否保留参数、公式、统计等动态能力
3. 不同方案分别对应什么开发与发布要求

---

## 二、核心结论

- 目前**尚未最终确定上线方案**，需由**陶扬**与**世豪**继续沟通拍板。  
- 若希望**保留参数修改、公式修改、使用统计**等能力，则需要依赖后端数据库，意味着要走**正式需求开发流程**。  
- 若接受只上线**纯静态前端页面**，则实施更快，但会失去管理员配置和动态调整能力。  
- 若考虑**腾讯云独立部署**，前提是**不接入行内后台系统、且不涉及敏感数据**，该方向仍需单独评估。

---

## 三、方案对比

### 方案一：RM 系统 / 行内正式开发方案

适用前提：**需要保留动态能力，或需要符合完整的行内生产发布流程。**


| 对比项                | 说明                   |
| ------------------ | -------------------- |
| **是否保留参数/公式/统计能力** | 保留                   |
| **是否需要后端开发**       | 需要                   |
| **开发复杂度**          | 较高                   |
| **发布要求**           | 须正式提需求，按行内规范开发、测试、发布 |
| **主要优点**           | 功能完整，符合正式生产规范        |
| **主要缺点**           | 周期较长，对后端资源依赖较强       |
| **当前结论**           | 如需保留动态能力，原则上应走该方案    |


### 方案二：纯静态前端页面方案

适用前提：**只保留页面展示与基础交互，不保留后台配置和统计能力。**


| 对比项                | 说明                                     |
| ------------------ | -------------------------------------- |
| **是否保留参数/公式/统计能力** | 不保留                                    |
| **是否需要后端开发**       | 不需要                                    |
| **开发复杂度**          | 较低                                     |
| **发布要求**           | 按公司前端框架规范开发，提交**文俊** Code Review，通过后发布 |
| **主要优点**           | 实施路径简单，上线更快                            |
| **主要缺点**           | 无法支持管理员参数配置、公式动态调整、使用统计                |
| **当前结论**           | 若业务接受功能收缩，可优先考虑该方案                     |


### 方案三：腾讯云独立部署方案（待评估）

适用前提：**希望团队自主部署，减少对行内系统排期依赖，且不涉及敏感数据与行内后台接入。**


| 对比项                | 说明                         |
| ------------------ | -------------------------- |
| **是否保留参数/公式/统计能力** | 保留                         |
| **开发方式**           | 可在外部环境使用 **CC** 进行开发与部署准备  |
| **是否需要后端开发**       | 需要                         |
| **开发复杂度**          | 整体开发成本和实现难度相对较低            |
| **发布要求**           | 需先确认是否允许采用腾讯云部署            |
| **主要优点**           | 可在外部环境较快推进开发，团队自主性更强，实施更灵活 |
| **主要缺点**           | 合规性、部署可行性仍需确认；且该方案此前曾被否决过  |
| **当前结论**           | 需后续专题确认                    |
